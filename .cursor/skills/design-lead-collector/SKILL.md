# Design Lead Collector

## Instructions

1. Start from the MVP: Landing Page, waitlist form, leads management database, analytics events, and admin Dashboard.
2. Keep the first version focused on 《聊著聊著，AI 就做完了》課程名單收集，不考慮多個課程或複雜行銷漏斗。
3. Use the `leads` and `analytics_events` tables as the main data models.
4. Treat trend charts, high-interest lead filtering, and AI copywriting pre-generation as optional or later-phase features unless requested.
5. Describe acceptance criteria as observable user actions and expected outcomes.

## Output Checklist

- Problem and user goal are clear.
- Inputs and validation rules (especially email and required fields) are explicit.
- UI regions (Landing Page, waitlist form, admin panel, Dashboard grid) are listed.
- Data fields and analytics event tracking are covered.
- Out-of-scope items (Won't Have) remain out of scope.

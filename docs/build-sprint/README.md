## Sprint 規劃總覽

本 MVP 分成 3 個 Sprint：

1. **Sprint 1**：假資料與基礎 Dashboard（US-01～US-05）
2. **Sprint 2**：互動篩選與學員列表（US-06～US-09）
3. **Sprint 3**：報表展示與 Google Sheet 串接規劃（US-10～US-12）

目標不是一次做成完整 BI 系統，而是先完成一條可展示、可驗收、可用於課程影片收束的最低成功路線。

後續（非本 MVP）：Sprint 4～6 見 [`docs/user-stories/README.md`](../user-stories/README.md)。

## Sprint 3：報表展示與 Google Sheet 串接規劃

### Sprint 目標

補上課程影片收束所需展示內容，並說明 MVP 如何從假資料版延伸成真實資料版。

完成後管理者可以：

- 用 Dashboard 做課程總結展示
- 用截圖或錄影呈現報表感
- 看到未來 Google Sheet／資料庫串接規劃
- 理解假資料版是驗證，不是終點

### 對應 User Stories

- [US-10](../user-stories/US-10-管理者可以看到報表展示區.md)
- [US-11](../user-stories/US-11-管理者可以看到Google-Sheet資料庫串接規劃.md)
- [US-12](../user-stories/US-12-管理者可以用Dashboard做課程總結展示.md)

### 實作任務

#### 1. 製作報表展示區

- [ ] 「本週課程數據摘要」或「課程學習成效摘要」
- [ ] 顯示關鍵指標摘要
- [ ] 版面適合截圖／錄影
- [ ] 不實作真正 PDF／CSV 匯出

#### 2. 製作後續串接說明區

- [ ] 標示目前資料來源是 mock data
- [ ] 說明下一步可串接 Google Sheet 或資料庫
- [ ] 說明替換資料來源後 Dashboard 結構不用重做
- [ ] 不深入 API 技術細節

#### 3. 製作課程總結展示段落

- [ ] 可用於影片結尾的說明
- [ ] 回扣「不是只看報名人數」
- [ ] 回扣「看有沒有開始學、跟上章節、完成作品」
- [ ] 回扣 Spec、US、Build Sprint 到作品完成
- [ ] 帶出畢業完成感

#### 4. 視覺與錄影整理

- [ ] 檢查第一屏、圖表文字、篩選操作、報表區
- [ ] 準備 1–2 個展示用篩選情境

### Sprint 3 驗收

- [ ] 可用於課程收束影片
- [ ] 報表展示區與後續串接規劃清楚
- [ ] 不會讓觀眾誤以為已完成真實資料串接
- [ ] 結尾有強烈畢業完成感
- [ ] 可錄製成 C-1 展示素材

---

## 建議展示順序

1. 打開 Dashboard，說明這是課程學員數據儀表板
2. 看 KPI：總學員數、已開始學習、平均完成進度、作品完成率
3. 看章節進度
4. 看作品狀態
5. 操作篩選
6. 展示報表區
7. 說明下一步：假資料 → Google Sheet／資料庫
8. 收束：從想法到產品的完整流程

### US-05：管理者可以看到作品完成狀態分布

**作為** 課程管理者
**我想要** 看到學員作品完成狀態
**以便** 知道學員是否真的把產品做出來

**輸入格式**：
- 假資料 projectStatus 欄位

**輸出格式**：
- 作品完成狀態圖表：人數與比例
- 狀態：尚未開始、Build Sprint 中、已完成 MVP、已發布

**驗收條件**：
- [ ] 圖表顯示作品狀態分布
- [ ] 狀態包含：尚未開始、Build Sprint 中、已完成 MVP、已發布
- [ ] 顯示每個狀態的人數
- [ ] 顯示每個狀態的比例
- [ ] 數據與學員列表一致

**依賴關係**：
- US-02；版面 US-01；與 US-06 列表一致性於 Sprint 2 可再驗證

**優先級**：P0
**相關功能**：Sprint 1｜作品狀態圖表

**設計稿來源**：
- 首頁精簡版：[`designs/dashboard-screen-1.pen`](../../designs/dashboard-screen-1.pen) → `作品完成狀態`（HTML：[`dashboard-screen-1.html`](../design/ref/dashboard-screen-1.html)）
- 詳情／解讀版（優先對齊）：[`designs/dashboard-screen-2.pen`](../../designs/dashboard-screen-2.pen) → `作品完成面板`＋教學說明（HTML：[`dashboard-screen-2.html`](../design/ref/dashboard-screen-2.html)）
- 實作約束：四狀態（尚未開始／Build Sprint 中／已完成 MVP／已發布）的人數、比例與視覺層級對齊設計稿；不新增「規劃中」。

**備註**：
- 呼應課程核心：不是只學 AI 工具，而是把作品做出來。本版本不使用「規劃中」。

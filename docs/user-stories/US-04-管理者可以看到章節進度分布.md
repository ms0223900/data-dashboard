### US-04：管理者可以看到章節進度分布

**作為** 課程管理者
**我想要** 看到各章節的完成分布
**以便** 知道學員目前主要完成到哪些階段

**輸入格式**：
- 假資料或 ChapterProgress 彙總
- 固定六個章節名稱

**輸出格式**：
- 章節進度圖表：各章節名稱與完成率或完成人數
- 手機版可閱讀

**驗收條件**：
- [ ] 圖表顯示每個章節名稱
- [ ] 圖表顯示每個章節完成率或完成人數
- [ ] 章節包含：想法收斂、Spec、User Stories、Build Sprint、驗收與修正、發布／延伸應用
- [ ] 數據與假資料一致
- [ ] 圖表在手機版仍可閱讀

**依賴關係**：
- US-02；版面 US-01

**優先級**：P0
**相關功能**：Sprint 1｜章節進度圖表

**設計稿來源**：
- 首頁精簡版：[`designs/dashboard-screen-1.pen`](../../designs/dashboard-screen-1.pen) → `Charts Row`／`章節進度分布`（HTML：[`dashboard-screen-1.html`](../design/ref/dashboard-screen-1.html)）
- 詳情／解讀版（優先對齊圖表互動與解讀）：[`designs/dashboard-screen-2.pen`](../../designs/dashboard-screen-2.pen) → `章節進度面板`＋`資料解讀提示`（HTML：[`dashboard-screen-2.html`](../design/ref/dashboard-screen-2.html)）
- 實作約束：六章節名稱、bar 排列與「Spec → Build Sprint 落差」解讀提示以 screen-2 為準；首頁可採較精簡呈現但資料同源。

**備註**：
- 不需要做「章節卡點」深度分析，本版本先呈現章節完成分布即可。

### US-02：系統內建一組課程學員假資料

**作為** 開發者
**我想要** 先使用假資料建立 Dashboard
**以便** 在不用串接真實資料的情況下驗證畫面與指標邏輯

**輸入格式**：
- Student 資料模型欄位定義（見 docs/spec.md §7）
- learningStatus／projectStatus 枚舉

**輸出格式**：
- 集中管理的假資料模組（≥20 筆）
- 涵蓋不同進度與不同作品狀態
- 方便後續替換為 Google Sheet／資料庫

**驗收條件**：
- [ ] 假資料至少包含 20 筆學員資料
- [ ] 每筆學員資料包含姓名、Email、完成進度、目前章節、學習狀態、作品狀態
- [ ] 假資料集中管理，方便後續替換
- [ ] 假資料包含不同進度與不同作品狀態
- [ ] 假資料不使用真實個資

**依賴關係**：
- 無

**優先級**：P0
**相關功能**：Sprint 1｜假資料

**備註**：
- 影片中可強調：先用假資料不是偷懶，而是先確認產品判斷邏輯。

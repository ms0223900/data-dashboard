## 依賴關係圖

```text
MVP
  US-01 (Dashboard 骨架)
  US-02 (假資料) ──┬──> US-03 (KPI)
                   ├──> US-04 (章節圖表)
                   ├──> US-05 (作品圖表)
                   └──> US-06 (學員列表) ──┬──> US-07 (學習狀態篩選)
                                          ├──> US-08 (作品狀態篩選)
                                          └──> US-09 (進度區間篩選)

  US-01 + US-02/03 ──> US-10 (報表展示)
  US-01 ─────────────> US-11 (串接規劃說明)
  US-01～US-11 ──────> US-12 (課程總結展示)

後續（非 MVP）
  MVP 完成 ──> US-13 ──> US-14
  US-06 ─────> US-16 ──> US-17
               US-15（週報）
  US-06 ─────> US-18
  US-04+06 ──> US-19
  US-18+19 ──> US-20
```

## 依賴說明

- **US-02** 是幾乎所有數據區塊的前置：沒有假資料就無法驗收 KPI／圖表／列表。
- **US-01** 提供承載區塊的頁面；可與 US-02 平行，但整合驗收需兩者皆完成。
- **US-07～09** 都掛在 **US-06** 上；彼此可平行實作。
- **US-12** 是展示收束故事，依賴 Sprint 1–3 主路徑可用。
- **US-13～20** 明確不列入本 MVP，僅作產品延伸地圖。

## 建議開發順序（Wave）

| Wave | 內容 | 可平行？ |
| --- | --- | --- |
| Wave 1 | US-01、US-02 | 是 |
| Wave 2 | US-03、US-04、US-05 | 是（皆依賴 US-02） |
| Wave 3 | US-06 | 否（篩選前置） |
| Wave 4 | US-07、US-08、US-09 | 是 |

### US-11：管理者可以看到 Google Sheet／資料庫串接規劃

**作為** 課程管理者
**我想要** 看到未來串接 Google Sheet 或資料庫的規劃
**以便** 知道假資料版 Dashboard 後續可以如何變成真實資料版

**輸入格式**：
- 產品文案／說明內容（無需 API key）

**輸出格式**：
- 「後續串接」說明區：標示假資料、可換 Sheet 或 DB
- 不安裝真實 API、不講資料工程細節

**驗收條件**：
- [ ] 頁面有「後續串接」說明區
- [ ] 說明目前資料來源是假資料
- [ ] 說明下一步可以把資料來源替換成 Google Sheet
- [ ] 說明也可以替換成資料庫
- [ ] 不需要實作真實 API 串接
- [ ] 不需要講底層資料工程細節

**依賴關係**：
- US-01

**優先級**：P0
**相關功能**：Sprint 3｜串接規劃說明

**設計稿來源**：
- **尚無獨立 Pencil 畫面**（現有 `dashboard-screen-1～3.pen` 未含「後續 Google Sheet／資料庫串接」專區）
- 文案與資訊架構暫依 [`docs/spec.md`](../spec.md) §9.1／§10 Flow 3
- 實作約束：須清楚標示目前為假資料、可替換為 Sheet 或 DB；不安裝真實 API。若後續補設計稿，以新稿覆蓋本節並同步更新本 US。

**備註**：
- 強調：先確認畫面與指標，下一步才串資料。
